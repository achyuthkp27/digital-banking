# Digital Banking

#PDS

A modern, high-performance marketing site showcasing the next generation of digital finance. Built with Next.js, React, Tailwind CSS, Framer Motion, and Three.js, and deployed as a fully static site to GitHub Pages.

## Overview

A beautifully animated, responsive, multilingual landing site for a Digital Banking platform. It features dynamic 3D/WebGL elements, micro-animations, and a polished UI. Content is available in English, Spanish, and French.

## Tech Stack

- **Framework**: Next.js 16 (App Router, static export via `output: 'export'`)
- **Language**: TypeScript (strict)
- **Styling**: Tailwind CSS v4 + CSS custom-property design tokens
- **Animations**: Framer Motion (respects `prefers-reduced-motion`)
- **3D Rendering**: Three.js & React Three Fiber (code-split, viewport-gated, skipped on low-end/mobile)
- **i18n**: next-intl (`en`, `es`, `fr`)
- **Icons**: Lucide React
- **Testing**: Jest + Testing Library (via `next/jest`)

## Getting Started

```bash
npm install
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) in your browser.

## Internationalization

Locales are defined in `src/i18n/routing.ts`. Translations live in `messages/{en,es,fr}.json` and must stay at key parity. Product copy is sourced from the `ProductPages` namespace; the canonical list of product slugs is `src/data/productContent.ts` (`productSlugs`) — used to drive routing, the sitemap, and the product illustration map so they cannot drift.

## Deployment

The site is statically exported and deployed to GitHub Pages under the `/digital-banking` base path.

- `basePath` / `assetPrefix` are applied only in production (`next.config.ts`).
- SEO metadata (`sitemap.ts`, `robots.ts`, `manifest.ts`) is generated by the App Router so URLs are basePath-aware — do not add static duplicates under `public/`.
- CI (`.github/workflows/nextjs.yml`) runs lint → typecheck → test → build, then deploys `out/` to Pages on push to `main`.

To point the product "Visit Site" CTAs at the real application, set `NEXT_PUBLIC_APP_URL` at build time (see `src/config/links.ts`).
